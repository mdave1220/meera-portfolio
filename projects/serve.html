THE BRIEF //

Serve is a social recipe app for people who discover food everywhere: TikTok, Instagram, blogs, group chats. But they lose recipes before they ever cook them. The gap isn't finding inspiration — it's closing the loop from "I saw this" to "I'm making this tonight."

DISCOVERY //

Ran 15+ semi-structured 1:1 interviews with college students and full-time workers. Top pain points: recipes trapped on social platforms, re-typing ingredients as friction, inspiration that never converts to a grocery run, and wanting to see what people they know actually cook. That pinned the core problem: recipe discovery and cooking inspiration, not another logging tool.

DEFINING THE MVP //

Three launch features: AI voice-to-text recipe input, cross-app recipe saving, and a real-time grocery stock checker. Scoped 4 social primitives (follows, likes, comments, DMs) so the product feels alive at launch. Built an LLM importer + Expo push notifications. Cut meal planning, nutrition tracking, and a web client.

PRE-LAUNCH VALIDATION //

Pressure-tested in Figma and early Expo builds with interview participants. Voice input moved up when users described typing with messy hands mid-recipe. Importer flow simplified when testers couldn't tell if a pasted URL had parsed. If a feature didn't survive a real weeknight cooking conversation, it didn't make the launch cut.

WHAT I OWNED //

End-to-end product ownership: interviews, MVP definition, design decisions, release cadence. On the build side — normalized Supabase Postgres schema, LLM importer pipeline, push notifications. Stayed close enough to implementation that scope reflected real constraints.
